## What Was Implemented

### Problem
In long-running agent shells, `--cwd /path/to/repo` had to be re-passed on every command. `SPINE_ROOT` solved this for single-repo sessions but required manual env management (`export`/`unset`) for multi-repo workflows, polluting the calling process environment. There was also no way to inspect what SPINE was targeting without running a full governance command.

### Solution: `spine target` command

Added `src/spine/cli/target_cmd.py` — a new `spine target` command that:

1. **Shows the currently resolved target repository** with its source annotation (`--cwd`, `SPINE_ROOT`, or `current directory`). Makes targeting state inspectable without running a full governance command.

2. **Documents the one-shot `SPINE_ROOT` pattern** in the command help:
   ```bash
   # Long-running shell: session default (set once)
   export SPINE_ROOT=/path/to/repo
   
   # One-shot: target a repo for one command only — no shell pollution
   SPINE_ROOT=/path/to/repo spine doctor
   SPINE_ROOT=/path/to/repo spine evidence add ...
   ```

3. **Supports `--json`** for machine-readable output — useful for agents verifying their targeting context.

4. **Preserves the full targeting contract** — no changes to `resolve_roots()` behavior or precedence. Contract: `--cwd` > `SPINE_ROOT` > cwd.

### Ergonomic improvement
The `SPINE_ROOT=/path spine <cmd>` one-shot pattern (POSIX scoped env var) was already supported but undocumented. It solves multi-repo use without shell pollution: the env var is scoped to a single command invocation, leaving the calling shell unaffected. This is now documented in `spine target --help`, `resolve_roots()` docstring, and `docs/SPINE_AGENT_SKILL.md`.
